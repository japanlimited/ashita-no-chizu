# あしたの地図 外部公開手順書

## 0. この手順書の目的

この手順書は、WebアプリMVP「あしたの地図」を外部公開するための作業を、初めてでも順番に進められるようにまとめたものです。

最初の公開先は **GitHub Pages** を推奨します。

理由:

- 今のアプリは `index.html` / `styles.css` / `app.js` だけで動く静的Webアプリ
- サーバーやデータベースをまだ使っていない
- LocalStorage保存なので、個人情報をサーバーに送らない
- 無料で試しやすい
- 公開URLをすぐ確認できる

## 2. 最初の公開方法

今回は、次の流れで進めます。

```text
Step 1. GitHubアカウントを用意する
Step 2. Gitを使える状態にする
Step 3. 公開用リポジトリを作る
Step 4. アプリのファイルをGitHubへアップロードする
Step 5. GitHub Pagesを有効化する
Step 6. 公開URLで表示確認する
Step 7. 修正したら再アップロードする
```

## 3. Step 1: GitHubアカウントを用意する

### 3.1 やること

1. GitHubにアクセスする
2. アカウントを作成する、またはログインする
3. ユーザー名を確認する

GitHub:

```text
https://github.com/
```

### 3.2 確認すること

あとで公開URLにユーザー名が使われます。

例:

```text
https://あなたのユーザー名.github.io/ashita-no-chizu/
```

### 3.3 今回確認できたユーザー名

今回のGitHubユーザー名は次の前提で進めます。

```text
japanlimited
```

そのため、公開URLは次の形になる予定です。

```text
https://japanlimited.github.io/ashita-no-chizu/
```

## 4. Step 2: Gitを使える状態にする

### 4.1 Gitが入っているか確認

PowerShellで次を実行します。

```powershell
git --version
```

バージョンが表示されればOKです。

例:

```text
git version 2.xx.x
```

### 4.2 Gitが入っていない場合

Git for Windowsをインストールします。

```text
https://git-scm.com/download/win
```

インストール後、PowerShellを開き直して、もう一度確認します。

```powershell
git --version
```

## 5. Step 3: GitHubで公開用リポジトリを作る

### 5.1 リポジトリ名

最初は次の名前を推奨します。

```text
ashita-no-chizu
```

### 5.2 GitHubでの操作

1. GitHub右上の `+` を押す
2. `New repository` を選ぶ
3. `Repository name` に `ashita-no-chizu` と入力する
4. 公開テストなら `Public` を選ぶ
5. `Add a README file` はオフのままにする
6. `Create repository` を押す

### 5.3 Public / Private の違い

```text
Public:
  誰でもコードを見られる
  GitHub Pages公開がわかりやすい

Private:
  コードは非公開
  GitHub Pagesの利用条件がプランや設定に影響する場合がある
```

最初の練習では `Public` が簡単です。

## 6. Step 4: アプリのファイルをGitHubへアップロードする

### 6.0 今回の注意

現在、親フォルダの `環境構築用` 側にGit管理があります。

公開したいのはアプリ本体だけなので、必ず次のフォルダに移動してから作業します。

```text
C:\Users\japan\OneDrive\Desktop\環境構築用\ashita-no-chizu
```

`環境構築用` フォルダ全体を公開しないように注意してください。

### 6.1 PowerShellでアプリフォルダへ移動

```powershell
cd C:\Users\japan\OneDrive\Desktop\環境構築用\ashita-no-chizu
```

### 6.2 Gitリポジトリとして初期化

```powershell
git init
```

### 6.3 ファイルを登録

```powershell
git add .
```

### 6.4 最初の保存履歴を作る

```powershell
git commit -m "Initial MVP"
```

### 6.5 ブランチ名をmainにする

```powershell
git branch -M main
```

### 6.6 GitHubのリポジトリと接続

今回はユーザー名 `japanlimited` として実行します。

```powershell
git remote add origin https://github.com/japanlimited/ashita-no-chizu.git
```

### 6.7 GitHubへアップロード

```powershell
git push -u origin main
```

ログインを求められた場合は、GitHubの案内に従って認証します。

## 7. Step 5: GitHub Pagesを有効化する

### 7.1 GitHub画面での操作

1. GitHubで `ashita-no-chizu` リポジトリを開く
2. 上部メニューの `Settings` を開く
3. 左メニューの `Pages` を開く
4. `Build and deployment` を探す
5. `Source` を `Deploy from a branch` にする
6. `Branch` を `main` にする
7. フォルダを `/ (root)` にする
8. `Save` を押す

### 7.2 待つ

公開には数十秒から数分かかることがあります。

## 8. Step 6: 公開URLで表示確認する

## 10. よくあるつまずき

### 10.1 公開URLを開いても404になる

確認すること:

- `Settings > Pages` で `main` と `/ (root)` が選ばれているか
- `index.html` がリポジトリ直下にあるか
- GitHub Pagesの反映待ちではないか

### 10.2 CSSが反映されない

確認すること:

- `styles.css` がGitHubにアップロードされているか
- `index.html` のCSS読み込みが `./styles.css` になっているか
- ブラウザを強制更新する

```text
Ctrl + F5
```

### 10.3 JavaScriptが動かない

確認すること:

- `app.js` がGitHubにアップロードされているか
- `index.html` のJS読み込みが `./app.js` になっているか
- ブラウザの開発者ツールでエラーを見る

### 10.4 ローカルでは動くが公開先で動かない

よくある原因:

- ファイル名の大文字・小文字が違う
- GitHubに一部ファイルをアップロードしていない
- ブラウザキャッシュが残っている

## 11. 現時点の公開に関する注意

### 11.1 AI APIについて

今のMVPは、OpenAI APIに直接つながっていません。

公開しても、ブラウザ内のローカル分析フォールバックで動きます。

### 11.2 OpenAI APIをつなぐ場合

将来OpenAI APIを使う場合、次の構成に変更します。

```text
ブラウザ
  ↓
自分のサーバー / Next.js API Route / Cloud Functions
  ↓
OpenAI API
```

やってはいけない構成:

```text
ブラウザ
  ↓
OpenAI API
```

理由:

- APIキーが利用者に見えてしまう
- 不正利用される可能性がある
- 利用料金が予期せず増える可能性がある
